### 🛡️ Aegis-X &nbsp;—&nbsp; *Flagship*

> An autonomous deepfake detection system where a locally-running LLM brain orchestrates **8 orthogonal forensic tools** — 5 physics-based (CPU), 3 transformer-based (GPU) — and explains its verdict in natural language grounded in specific tool evidence.

<table>
<tr>
<td width="50%" valign="top">

**Why it's different**

Most detectors are CNNs trained on a specific generator.  
The moment a new generator ships, the detector breaks.

Aegis-X replaces generator-specific fingerprints with **physics laws, frequency mathematics, and generator-agnostic transformers** — signals that cannot be invalidated by a new release.

```
Input
  │
  ├─ CPU Phase  (zero VRAM, zero training data)
  │   ├─ C2PA provenance gate
  │   ├─ rPPG biological liveness (POS algorithm)
  │   ├─ DCT double-quantization analysis
  │   ├─ 7-point anthropometric consistency
  │   ├─ Shape-from-shading physics
  │   └─ Corneal reflection consistency
  │
  │   ── CONFIDENCE GATE (>0.85 → skip GPU entirely) ──
  │
  ├─ GPU Phase  (sequential, VRAM-efficient)
  │   ├─ CLIP adapter  (ViT-B/32 frozen + 993K params)
  │   ├─ SBI detector  (EfficientNet-B4, CVPR '22)
  │   └─ F3Net         (ResNet-50 + FAD, ECCV '20)
  │
  └─ Phi-3 Mini 3.8B → Verdict + Natural Language Reasoning
```

</td>
<td width="50%" valign="top">

**Technical highlights**

- **Signal orthogonality** — CPU tools catch what neural nets structurally cannot learn; GPU tools generalize across generators
- **Confidence gate** — skips all GPU tools when CPU phase confidence > 0.85, saving 40–80% compute
- **LSE pooling** — learnable smooth-max aggregation over 6 CLIP patch scores with interpretable cross-patch attention heatmap
- **Temporal latent jitter** — detects generative video (Sora) via inter-frame cosine variance in CLIP latent space, zero extra VRAM
- **Abstention contract** — crashed tools vote `(0.0, 0.0)`, never as confident REAL
- **100% local** — GDPR-ready, air-gap compatible, no biometric data leaves the machine

**Stack**
